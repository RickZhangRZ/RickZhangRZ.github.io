一篇搞懂排序

2019-11-28 排序 0 评论 字数统计: 1.2k(字) 阅读时长: 5(分)

注:以下排序均为升序

一. 冒泡排序(Bubble Sort)

思路:

  • 从头开始比较每一对相邻元素,如果第一个比第二个大,就交换它们的位置,执行完一轮后,最末尾那个元素就是最大的元素
  • 忽略第一步中找到的最大元素,重复执行第一步,直到全部元素有序
  1. 基本实现
  2. 优化1
    如果序列已经完全有序了,可以提前终止冒泡
  3. 优化3
    如果序列尾部已经局部有序了,可以记录最后一次交换的位置,减少比较次数

总结
对于以上三种冒泡排序算法,其时间复杂度均为O(n的平方),那么后面两种优化相对于第一种有什么区别,主要区别在于后面两种较少了比较的次数,减少了运行的时间

二. 选择排序(Selection Sort)

思路:

  • 从序列中找出最大的那个元素,然后与最末尾的元素交换位置,执行完一轮后,最末尾的那个元素就是最大的元素
  • 忽略第一步曾经找到的最大元素,重复步骤1
  1. 基本实现
  2. 代码优化
    实质上选择排序的优化就是堆排序了

总结
时间复杂度:O(n的平方)

插入排序(Insertion Sort)

思路:

插入排序就类似于扑克牌的排序,执行流程为:

  • 在执行过程中,插入排序讲序列分为两个部分,头部是已经排好序的,尾部是待排序的
  • 从头开始扫描每一个元素,每当扫描到一个元素,就将它插入到头部合适的位置,使得头部数据依然保持有序
  1. 基本实现
  2. 优化1
    优化的思路在于每次在往前遍历的时候,每次遇到比目标值大,就要进行交换,如果将交换改为挪动,则可以达到一定的优化
  3. 优化2
    插入排序的优化主要在于选择合适的位置来插入,那在选择合适位置的思路上可以利用二分搜索来进行选择

    二分搜索

    二分搜索是在已排序好的元素进行操作的,插入排序刚好符合这特征
    二分搜索思路:
  • 假设在[begin,end)范围内搜索某个元素v,mid == (begin + end) / 2
  • 如果v < m,去[begin,mid]范围内二分搜索
  • 如果v >= m(保证算法的稳定性),去[mid + 1, end]范围内二分搜索
  • 如果v == m, 直接返回mid
